📦 PickUp RangerPickUp Ranger℠

Einmal-Links für GLS-Abholungen erzeugen, kopieren und Status verfolgen. Generate one-time GLS pickup links, copy them and track status.

AblaufWorkflow

  1. CS erzeugt einen Einmal-Link für den Kunden (Name, E-Mail, Ticket, Grund).CS creates a one-time link for the customer (name, email, ticket, reason).
  2. Der Link wird via Zendesk-Antwort verschickt — nie direkt vom Kunden anforderbar.The link is sent via Zendesk reply — never user-requestable.
  3. Der Kunde öffnet den Link, füllt Adresse und Zeitfenster aus und bestätigt.The customer opens the link, fills in address and slot, and confirms.
  4. Server prüft Token (gültig, nicht abgelaufen, nicht eingelöst) und ruft die GLS-API.Server validates the token (valid, not expired, not used) and calls the GLS API.
  5. Nach Buchung wechselt der Link in den Tracking-Modus — keine zweite Buchung mehr möglich.After booking the link switches to tracking mode — no second booking possible.

SicherheitSecurity

  • Token mit hoher Entropie, serverseitig gespeichert, gebunden an Ticket & Kunde.High-entropy token, server-side, bound to ticket & customer.
  • Single-Use erzwungen (Status-Maschine: ISSUED → CONFIRMED → PICKED_UP → COMPLETED).Single-use enforced (state machine: ISSUED → CONFIRMED → PICKED_UP → COMPLETED).
  • Ablaufzeit (Standard 7 Tage), Rate-Limit pro IP, Audit-Log.Expiry (default 7 days), per-IP rate limit, audit log.
  • Kein Login-Konto nötig — keine Wiederverwendbarkeit, keine Passwort-Risiken.No customer account needed — no reuse, no password risk.
