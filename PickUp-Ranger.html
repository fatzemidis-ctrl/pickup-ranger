Sichere Einmal-Links für GLS-Abholungen. Secure one-time links for GLS pickups.

Erstelle persönliche Abhol-Links für einzelne Kunden, kopiere sie in deine Zendesk-Antwort, und verfolge den Status jeder Abholung — ohne Kunden-Login, ohne mehrfache Nutzung, ohne Risiko. Issue personal pickup links for individual customers, copy them into your Zendesk reply, and track the status of each pickup — no customer login, no reuse, no risk.

So funktioniert esHow it works

AblaufWorkflow

  1. CS erzeugt einen Einmal-Link für den Kunden (Name, E-Mail, Ticket, Grund).CS issues a one-time link for the customer (name, email, ticket, reason).
  2. Der Link wird über die Zendesk-Antwort verschickt — niemals direkt vom Kunden anforderbar.The link is sent via Zendesk reply — never user-requestable.
  3. Der Kunde öffnet den Link, füllt Adresse und Zeitfenster aus, bestätigt.The customer opens the link, fills in address and time window, confirms.
  4. Server prüft Token (gültig, nicht abgelaufen, nicht eingelöst) und ruft die GLS-API.Server validates the token (valid, not expired, not used) and calls the GLS API.
  5. Nach Buchung schaltet der Link auf Tracking um — keine zweite Buchung möglich.After booking the link switches to tracking mode — no second booking possible.

SicherheitSecurity

  • Hochentropie-Token, serverseitig gespeichert, an Ticket + Kunde gebunden.High-entropy token, stored server-side, bound to ticket + customer.
  • Single-Use über Status-Maschine erzwungen: ISSUED → CONFIRMED → PICKED_UP → COMPLETED.Single-use enforced via state machine: ISSUED → CONFIRMED → PICKED_UP → COMPLETED.
  • Standardlaufzeit 7 Tage, Rate-Limit pro IP, Audit-Log.Default 7-day expiry, per-IP rate limit, audit log.
  • Kein Kunden-Account → kein Passwort-Risiko, keine Mehrfach-Nutzung.No customer account → no password risk, no reuse.
